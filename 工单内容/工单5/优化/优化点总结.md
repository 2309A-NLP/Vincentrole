# 工单5 优化点总结

> 工单编号: 人工智能NLP-RAG-Query理解优化任务

---

## ✅ 已实现的优化（与工单4对比）

### 1. Milvus 向量数据库（新增）

| 对比项 | 工单4 | 工单5 |
|--------|-------|-------|
| 向量存储 | FAISS 本地文件 | Milvus 分布式数据库 |
| 持久化 | 文件系统 | 独立服务 + 数据持久化 |
| 高并发 | 单进程 | 支持多客户端并发检索 |
| 可扩展性 | 受限于单机内存 | 支持分布式分片 |
| 切换方式 | 固定 FAISS | `VECTOR_STORE_BACKEND = "faiss" / "milvus"` 双后端可切换 |

### 2. 多轮对话（新增）

| 功能 | 说明 |
|------|------|
| 历史保留 | 最近 3 轮对话记录 |
| 上下文窗口 | 历史文本上限 2000 字符 |
| 指代消解 | 自动解析"他/这个公司/那/上述"等指代 |
| 对话增强 | 历史上下文拼接后送入 LLM |
| Prompt 注入 | 在系统 Prompt 中加入 "请结合对话历史理解当前问题" |

**对话历史格式：**
```
【对话历史】
User: 武汉兴图新科注册资本是多少？
Assistant: 5,520.00万元
User: 它的法定代表人是谁？
Assistant: 程家明
【以上是对话历史，请结合上下文理解当前问题】
```

### 3. 关键词兜底检索（新增）

| 功能 | 说明 |
|------|------|
| 触发条件 | 向量检索返回 0 条时自动触发 |
| 检索方式 | BM25 关键词匹配 |
| 扩展数量 | 最多 +5 条兜底结果 |
| 应用场景 | Milvus 集合未加载 / 维度不匹配 / 向量检索异常时保底 |

### 4. 图像类型分类（优化）

| 分类 | 提取数量 | 说明 |
|------|---------|------|
| 示意图 | 20 | 技术图纸、结构图 |
| 流程图 | 15 | 业务流程、数据流 |
| 组织结构图 | 6 | 公司架构、部门关系 |
| 柱状图 | 2 | 数据对比图表 |

### 5. 多轮 Prompt 增强

```
用户第2轮提问"它的法定代表人是谁？" 
  → 指代消解 + 历史上下文: "武汉兴图新科的法定代表人是谁？"
  → 检索更精准 → LLM 答案正确
```

## ❌ 仍存在的问题

| 问题 | 原因 | 建议 |
|-----|------|------|
| pymilvus 3.0 API 不兼容 | `IndexParams` 构造方式改变，当前切回 FAISS | 升级代码适配新版 MilvusClient API |
| 图像多模态回退耗时 | 每轮尝试 3 张图，VL 调用慢 | 默认关闭图像提取，按需开启 |
| 指代消解规则简单 | 基于正则的简单匹配，不支持复杂指代 | 可引入 LLM 进行指代消解 |

## 🔜 下一步优化建议

1. **Milvus 索引修复** → 适配 pymilvus 3.0 API，启用真实向量检索
2. **指代消解增强** → 引入 LLM 或深度学习模型
3. **多轮对话截断优化** → 按 token 而非字符截断，避免切碎语义
4. **对话历史可视化** → 前端展示对话轮次和历史引用
