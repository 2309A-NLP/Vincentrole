## 1. 软件简介

PDF智能问答系统是一款基于RAG（检索增强生成）技术的文档问答工具。您可以上传PDF文档（如招股说明书、年报、研究报告等），然后通过自然语言提问，系统会从文档中检索相关信息并生成准确的回答。

### 核心功能

- 上传PDF文档，自动解析文本内容
- 通过自然语言提问，系统从文档中寻找答案
- 展示答案的来源页码
- 支持RAG结果 vs 纯LLM结果对比
- 内置测试问题，快速验证系统效果

## 2. 环境准备

### 2.1 安装Python

确保您的电脑上已安装 Python 3.9 或更高版本。

验证方法：打开终端（命令提示符），输入：

```bash
python --version
```

如果显示 `Python 3.9.x` 或更高版本，说明已安装。

### 2.2 安装依赖

打开终端，进入项目目录：

```bash
cd ~/Desktop/PDF-RAG-问答系统
pip install -r requirements.txt
```

安装过程可能需要 5~10 分钟，请保持网络畅通。

## 3. 启动系统

### 3.1 启动方式

在终端中执行：

```bash
cd ~/Desktop/PDF-RAG-问答系统
streamlit run app/ui.py
```

或：

```bash
python run.py
```

### 3.2 访问系统

启动后，终端会显示类似以下信息：

```
You can now view your Streamlit app in your browser.
Local URL: http://localhost:8501
```

自动打开浏览器，访问 http://localhost:8501 即可进入系统。

## 4. 使用指南

### 4.1 加载PDF文档

#### 方式一：使用默认测试文档

系统内置了"招股说明书1.pdf"作为测试文档，只需在左侧侧边栏勾选"使用默认测试文档"，然后点击"加载文档"按钮即可。

#### 方式二：上传自己的PDF

1. 点击左侧的"选择PDF文件"按钮
2. 选择电脑上的PDF文档（支持任意PDF）
3. 取消勾选"使用默认测试文档"
4. 点击"加载文档"

> **注意**：加载大型PDF（200页以上）可能需要 1~3 分钟，请耐心等待。

### 4.2 配置LLM（可选）

在左侧"LLM配置"区域，您可以：

1. **选择提供商**：
   - `openai` - 使用 OpenAI 的 GPT 模型（需API密钥）
   - `deepseek` - 使用 DeepSeek 模型（国内可直接访问，推荐）
   - `ollama` - 使用本地部署的 Ollama 模型

2. **输入API密钥**：如选择 OpenAI 或 DeepSeek，需输入对应的 API Key

3. **设置API地址**：如需使用代理或自部署服务，可修改API地址

4. **选择模型名称**：填写具体的模型名称

配置完成后点击"更新LLM配置"按钮保存。

### 4.3 提问

在页面底部的输入框中输入问题，按 Enter 键提交。

**示例问题**：
- 公司注册资本是多少？
- 报告期内军用领域收入占比是多少？
- 公司参与制定了哪个技术标准？
- 本次发行的保荐机构是哪家？
- 公司面临的主要风险有哪些？

### 4.4 查看答案

系统会展示：
- 答案内容
- 来源页码（从文档的哪几页找到的信息）
- 回答耗时

点击"查看检索详情"可以展开查看：
- Query分析结果（识别到的意图和实体）
- 检索到的相关文档片段

### 4.5 对比模式

在左侧开启"对比模式"后，每次提问会同时展示：
- **RAG回答**：基于检索到的文档内容生成
- **纯LLM回答**：仅靠大模型自身知识回答

通过对比可以直观感受 RAG 技术的优势——它让答案"有据可查"。

### 4.6 快速测试问题

左侧侧边栏提供了"快速测试问题"区域，包含工单规定的测试问题。点击任一问题即可自动填入并提交，快速验证系统效果。

## 6. 常见问题

### Q: 加载PDF时提示"文件不存在"？
A: 确保PDF文件路径正确。使用默认测试文档时，确保文件位于正确位置。

### Q: 回答内容为空或不准确？
A: 可能原因：
1. LLM配置不正确（API密钥无效或提供商不可用）
2. 问题太过宽泛，尝试更具体的问题
3. PDF文档中确实不包含相关信息

### Q: 检索耗时较长？
A: 首次加载PDF时需要构建索引，后续问答快速。大型PDF（500页+）的解析时间为 2~5分钟。

### Q: 支持哪些PDF格式？
A: 支持标准PDF文档。扫描版PDF（图片形式）需要使用OCR工具预处理。

### Q: 如何更换Embedding模型？
A: 编辑 `config.py` 中的 `EMBEDDING_CONFIG` 的"模型名称"字段，改为 HuggingFace 上的其他模型名。

## 7. 注意事项

1. **首次运行**会自动下载 embedding 模型（约 1.3GB），请确保网络通畅
2. **LLM API密钥**不会保存在代码中，每次启动需在界面中重新填写，或设置环境变量
3. **清空缓存**按钮会删除已构建的向量索引，下次加载PDF时会重新构建
4. 如有任何问题，请检查终端输出的错误信息
