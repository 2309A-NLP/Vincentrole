## 1. 系统概述

本系统是一个基于 RAG（Retrieval-Augmented Generation，检索增强生成）技术的 PDF 文档问答系统。用户上传 PDF 文档后，系统自动解析文档内容，构建向量知识库，然后基于用户问题从文档中检索相关信息，并通过大语言模型（LLM）生成准确、简洁的回答。

### 1.1 核心能力

- PDF文档解析（文本提取、分块处理）
- 语义向量化与高效检索（FAISS向量索引）
- 智能Query理解（意图识别、实体抽取、查询重写）
- LLM驱动答案生成
- RAG结果与纯LLM结果对比分析
- 直观的Web交互界面（Streamlit）

## 3. 模块说明

### 3.1 PDF解析模块 (pdf_parser/)

**parser.py** - PDF文本提取
- 使用 pymupdf 引擎（自动降级到 pypdf）
- 按页提取文本，附带页码元数据
- 自动清洗文本（去除多余空格、空行等）

**chunker.py** - 智能文本分块
- 保持段落和句子完整性
- 支持自定义块大小和重叠长度
- 超长段落自动拆分（按句号）
- 相邻块之间添加重叠内容，避免上下文断裂

### 3.2 知识库模块 (knowledge_base/)

**embeddings.py** - 文本向量化
- 使用 sentence-transformers（默认 BAAI/bge-base-zh-v1.5）
- 支持 BGE 模型的 query 前缀处理
- 向量 L2 归一化（确保余弦距离计算正确）
- 缓存机制加速重复文本编码

**vector_store.py** - 向量存储
- 基于 FAISS IndexFlatIP（内积=余弦相似度）
- 支持 构建/检索/保存/加载 完整生命周期
- 结果按相关性分数排序，支持阈值过滤

### 3.3 QA引擎模块 (qa_engine/)

**query_understanding.py** - Query理解
- 意图识别：6类金融文档常见意图（财务数据、公司信息、风险因素、行业分析、募资用途、股权结构）
- 实体抽取：公司名、金额、年份、比例等金融实体
- 查询重写：移除疑问词，提高检索匹配度

**retriever.py** - 语义检索
- 串联 Query理解 -> 向量化 -> FAISS检索 -> 结果格式化
- 自动截断超长上下文

**generator.py** - LLM答案生成
- 支持 OpenAI、DeepSeek、Ollama 三种后端
- 自定义系统提示词
- 纯LLM生成模式（用于对比实验）

**orchestrator.py** - 系统控制器
- 串联完整工作流：PDF解析 -> 分块 -> 向量化 -> 索引 -> 检索 -> 生成
- 支持 RAG vs 纯LLM 对比模式
- 索引缓存机制，避免重复构建

### 3.4 评估模块 (evaluation/)

**test_questions.py** - 测试问题集
- 工单规定的10个测试问题
- 4个扩展测试问题
- 按类别筛选

**evaluate.py** - 评估器
- 批量评估全部测试问题
- 统计：命中率、平均耗时、检索成功率
- RAG vs 纯LLM 对比数据

### 3.5 Web界面 (app/)

**ui.py** - Streamlit用户界面
- PDF上传 / 默认测试文档加载
- LLM配置面板（提供商、API密钥、模型选择）
- 普通问答模式 & 对比模式切换
- 历史对话记录
- 检索详情展开查看

## 4. 技术选型

| 组件 | 技术选型 | 选型理由 |
|------|---------|---------|
| 向量嵌入 | BAAI/bge-base-zh-v1.5 | 中文语义理解优秀，768维，兼顾效果和性能 |
| 向量存储 | FAISS (IndexFlatIP) | 精确检索，CPU可运行，轻量级 |
| PDF解析 | PyMuPDF + pypdf | 中文PDF支持好，双引擎自动降级 |
| Web界面 | Streamlit | 快速搭建交互界面，生态丰富 |
| LLM接入 | OpenAI/DeeplSeek/Ollama | 灵活切换，国内可用DeepSeek |
| Embedding框架 | sentence-transformers | 成熟稳定，模型丰富 |

## 6. 配置说明

配置文件 `config.py` 包含所有可调参数：

- **PDF_CONFIG**: PDF解析相关配置
- **CHUNK_CONFIG**: 分块大小（默认500字符）、重叠（100字符）
- **EMBEDDING_CONFIG**: 嵌入模型名称、设备、归一化
- **VECTOR_STORE_CONFIG**: 索引和元数据存储路径
- **RETRIEVAL_CONFIG**: 检索数量（默认5）、相关性阈值（0.3）
- **LLM_CONFIG**: 提供商、模型、API地址/密钥、温度、最大Token数
- **UI_CONFIG**: 页面标题、布局

## 7. 部署说明

### 环境要求
- Python 3.9+
- 内存: 4GB+（加载 embedding 模型）
- 磁盘: 1GB+（存放模型缓存）

### 安装步骤

```bash
# 1. 进入项目目录
cd PDF-RAG-问答系统

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置LLM（编辑 config.py 或通过界面配置）

# 4. 启动
streamlit run app/ui.py
```

### 注意事项
- 首次运行会自动下载 embedding 模型（约 1.3GB）
- LLM API密钥需自行配置（支持 OpenAI / DeepSeek / Ollama）
- 大型PDF（500+页）的解析和索引构建可能需要 2~5 分钟

## 8. 扩展性设计

本系统采用模块化设计，每个组件可通过接口替换：

- **PDF解析器**: 实现新的解析器继承 PDFParser
- **嵌入模型**: 更换 EMBEDDING_CONFIG 中的模型名称
- **向量存储**: 可切换为 Chroma / Milvus 等其他向量库
- **LLM后端**: 通过 generator.py 添加新的 provider
- **Query理解**: 扩展 INTENT_PATTERNS 字典支持更多意图类别
