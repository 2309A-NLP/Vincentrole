# 工单11 测试报告

## 一、测试概述

本次测试验证 Embedding 模型微调效果，以及微调模型在 RAG 系统中的表现。测试分为两部分：
1. **离线评估**：对比微调前后模型的 Embedding 相似度分数
2. **在线评估**：将微调模型集成到 RAG 系统，对比检索准确率

---

## 二、离线评估结果

### 2.1 数据概览

| 项目 | 数值 |
|------|------|
| 数据来源 | 9家上市公司年报PDF（ccf_competition 2） |
| 总 chunks 数 | 5,747 |
| 生成问答对 | 825 |
| 训练样本 | 1,320 |
| 验证样本 | 330 |

### 2.2 训练参数

| 参数 | 值 |
|------|-----|
| 基础模型 | BAAI/bge-base-zh-v1.5 |
| 框架 | sentence-transformers |
| 损失函数 | CosineSimilarityLoss |
| Epochs | 3 |
| Batch size | 16 |
| Learning rate | 2e-5 |
| 设备 | Apple M1 (CPU) |
| 训练耗时 | ~5分钟（3个epoch） |

### 2.3 评分对比

| 指标 | 微调前 | 微调后 | 提升 |
|------|--------|--------|------|
| Cosine Similarity Score | 0.8279 | 0.8660 | **+4.60%** |

### 2.4 Top-K 命中对比（A/B测试）

| 指标 | bge-base（原版） | finetuned-bge-base | 结果 |
|------|----------------|-------------------|------|
| Top-1 平均分 | 0.455 | **0.635** | **+39%** |
| Top-3 命中率 | 6/8 | **7/8** | ✅ |
| Top-5 命中率 | 7/8 | **8/8** | ✅ |
| 8题综合胜出 | - | **8/8** | ✅ 全胜 |

> 测试 8 道金融领域问题，微调模型在所有问题上的 Top-1 分数均高于原版模型。

## 四、RAG 系统集成测试

### 4.1 集成方式

微调模型通过修改 `config.py` 中的 `EMBEDDING_CONFIG` 路径集成到 RAG 系统：

```python
EMBEDDING_CONFIG = {
    "模型名称": "finetuned-bge-base-zh-v1.5",
    "模型路径": "./embed_finetune/finetuned-bge-base-zh-v1.5",
    "设备": "cpu",
    "归一化": True,
    "维度": 768,
}
```

## 五、问题与改进

### 5.1 已知问题

| 问题 | 影响 | 状态 |
|------|------|------|
| 训练数据有限（825对） | 泛化能力受限 | 待扩展 |
| 仅使用 CosineSimilarityLoss | 未尝试 TripletLoss | 待对比 |
| 答案提取依赖正则匹配 | 部分答案不准确 | 待优化 |
| 仅覆盖金融领域 | 跨领域泛化未验证 | 待测试 |

### 5.2 改进建议

1. **数据增强** → 用 LLM 重新生成问答对，提高数据质量
2. **多损失函数对比** → 尝试 TripletLoss / MultipleNegativesRankingLoss
3. **跨领域测试** → 在招股说明书数据集上验证泛化能力
4. **集成到 RAG 全流程** → 完成端到端微调→集成→评估闭环
