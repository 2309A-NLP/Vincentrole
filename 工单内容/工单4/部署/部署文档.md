## 3. 部署步骤

### 3.1 安装依赖

```bash
cd 工单4/研发
pip install -r requirements.txt
```

**依赖清单：**

| 包名 | 用途 |
|------|------|
| streamlit | Web 前端框架 |
| pymupdf (fitz) | PDF 文本提取 + 图像提取 |
| faiss-cpu | 向量索引与检索 |
| sentence-transformers | BGE Embedding 模型 |
| numpy | 数值计算 |
| rank-bm25 | BM25 关键词检索 |
| requests | LLM API 调用 |

### 3.2 配置 API

编辑 `config.py`：

```python
LLM_CONFIG = {
    "提供商": "custom",                    # UI 只支持 deepseek/openai/ollama/custom
    "模型": "qwen-plus",
    "API地址": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "API密钥": "sk-your-api-key-here",
    "最大Token数": 1024,
    "温度": 0.1,
    "超时": 15,
    "重试次数": 2,
}
```

> 提供商设为 `custom`，API 地址指向通义千问的 OpenAI 兼容端点。

### 3.3 配置图像模块（可选）

```python
IMAGE_CONFIG = {
    "启用图像提取": False,             # 设为 True 开启图像提取（建议首次测试为 False）
    "提取最小尺寸": (100, 100),        # 过滤装饰性小图
    "保存格式": "png",
    "输出目录": os.path.join(KB_DIR, "extracted_images"),
    "最大图像数": 200,
    "图像质量": 95,
    "多模态模型": "clip_vit_b_32",      # clip 或 bge_text_fallback
    "图像描述方式": "提取上下文文本",    # 上下文文本 / 模型生成 / 关闭
    "上下文窗口": 1500,
    "图像检索权重": 0.3,
    "图像指令命中提升": 0.4,
    "检索图像上限": 3,

```

> **注意：** 国内网络无法下载 CLIP 模型，图像向量会回退为 BGE 文本编码。如需精确图像检索，请离线下载 CLIP 模型。

## 4. 首次使用流程

```
1. 打开浏览器 → http://localhost:端口
2. 系统自动加载 PDF
3. 如未缓存索引，自动解析 → 分块 → 向量化 → 构建索引（约 1 分钟）
4. 在输入框输入问题
5. 系统执行：文本检索 → 图像感知融合 → LLM生成 → 返回结果
```

### 首次加载耗时参考

| 阶段 | 耗时 |
|------|------|
| PDF 解析 + 表格检测（548页） | ~15s |
| 语义分块（2132个chunk） | ~5s |
| BGE 向量化 | ~30s |
| 图像提取（可选） | ~10s |
| 构建索引 | ~2s |
| **总计** | **~50-70s** |

## 6. 常见问题

### Q1: 报错 "ValueError: 'dashscope' is not in list"

**原因：** 提供商设为 `dashscope`，但 UI 只支持 `deepseek/openai/ollama/custom`。
**解决：** 改为 `custom`，API 地址指向通义千问。

### Q2: CLIP 模型无法加载

**原因：** 国内网络无法下载 huggingface 模型。
**解决：** 配置中已默认使用 BGE 文本回退方案，无需 CLIP。

### Q3: 多模态回退超时

**原因：** VL API 调用慢或图片无有效信息。
**解决：** 关闭图像提取（`启用图像提取: False`）或关闭图像描述（`图像描述方式: "关闭"`）。

### Q4: 如何重新构建索引？

```bash
rm -f data/vector_index.faiss data/chunk_metadata.json data/chunk_metadata_bm25.pkl
rm -rf data/extracted_images/
```
重启系统自动重建。

## 7. 与工单3 的区别

| 功能 | 工单3 | 工单4 |
|------|-------|-------|
| 文档数量 | 多文档 | 多文档 |
| 表格解析 | 结构化表格 | 结构化表格 + 图像提取 |
| 检索增强 | 表格感知（列名+标题） | 表格 + 图像双感知 |
| 图像提取 | 无 | PyMuPDF 提取，上下文描述 |
| 查询扩展 | 表格相关 | 表格 + 图像双扩展（5组） |
| 跨模态融合 | 无 | 文本×0.7 + 图像×0.3 |
| 多模态回退 | 无 | 可选 VL 回退（默认关闭） |
| API | 通义千问 | 通义千问 |
