# 工单4 优化点总结

> 工单编号: 人工智能NLP-RAG-04-图像解析优化

---

## ✅ 已实现的优化（与工单3对比）

### 1. 图像提取与存储（新增）

| 对比项 | 工单3 | 工单4 |
|--------|-------|-------|
| PDF 图像提取 | 无 | PyMuPDF 提取 PDF 页面图片 |
| 输出格式 | 纯文本+表格 | 文本 + 表格 + PNG 图片 |
| 图像过滤 | 无 | 过滤装饰性小图（<100×100） |
| 图像上限 | 无 | 单文档最多 200 张 |

### 2. 图像上下文描述（新增）

| 功能 | 说明 |
|------|------|
| 上下文窗口 | 提取图像前后各 1500 字符作为图像描述 |
| 存储格式 | `extracted_images/` 目录 + `image_metadata.json` 索引 |
| 描述文本 | 图像保存时关联上下文文字，用于纯文本检索 |

### 3. 图像感知查询扩展（新增）

```
查询"组织结构图" → 扩展为 ["组织结构图", "组织架构图", "销售部", "部门构成", ...]
查询"IC市场" → 扩展为 ["IC市场", "集成电路市场", "半导体市场", "增长率", ...]
查询"增长率" → 扩展为 ["增长率", "增长速度", "占比", "下滑", ...]
查询"图" → 扩展为 ["图", "图表", "流程图", "饼图", "柱状图", ...]
```

共 5 组图像相关的同义扩展，覆盖招股说明书中常见的图表类型。

### 4. 跨模态融合检索

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `图像权重提升` | 0.3 | 图像类型 chunk 额外加分 |
| `图像描述匹配提升` | 0.4 | 查询词匹配到图像描述时加分 |
| `检索数量` | 8（从 5 扩到 8） | 扩大检索覆盖范围 |
| `重排序数量` | 10（从 5 扩到 10） | 配合扩大后的检索数量 |

**检索融合公式：**
```
最终分 = 文本检索分 × 0.7 + 图像检索分 × 0.3
```

### 5. 多模态回退机制（新增）

| 步骤 | 说明 |
|------|------|
| ① 文本检索 | FAISS + BM25 + RRF 融合 |
| ② 图像匹配 | 命中图像指令时提升图像权重 |
| ③ 多模态回退 | 前 3 张匹配图片尝试 VL 问答 |
| ④ 文字回退 | 图片均无效时，回归纯文本 LLM 生成 |

### 6. 分块参数调优（本次优化）

| 参数 | 之前 | 之后 | 效果 |
|------|------|------|------|
| `分块大小` | 800 | 500 | 表格数据更集中，减少噪声 |
| `分块重叠` | 150 | 80 | 降低重复片段 |
| chunk 数 | 8106 | 2132 | chunk 更精炼，单条信息密度更高 |

## ❌ 仍存在的问题

| 问题 | 原因 | 建议 |
|-----|------|------|
| CLIP 模型不可用 | 国内网络无法下载 CLIP 权重 | 使用 BGE 文本回退方案，精度有损 |
| 多模态回退耗时长 | VL API 调用慢，且通常无效 | 当前已关闭，改为纯文本模式 |
| 图像描述依赖上下文 | 没有真正的视觉理解，仅靠周围文字 | 当 CLIP 可用时切换到真实视觉向量 |
| API Key 需手动配置 | 硬编码在 config.py | 改为环境变量或前端可配 |

4. **重排器集成** → 增加跨模态重排模型，优化图文混合排序
