# 工单3 优化点总结

> 工单编号: 人工智能NLP-RAG-PDF文档的表格解析及检索优化

---

## ✅ 已实现的优化（与工单2对比）

### 1. 多文档支持（新增）

| 对比项 | 工单2（单文档） | 工单3（多文档） |
|--------|---------------|---------------|
| PDF 数量 | 单本招股说明书 | 同时加载两本招股书 |
| 配置方式 | `DEFAULT_PDF_PATH` 单路径 | `DEFAULT_PDFS` 路径列表 |
| 索引策略 | 单文档单索引 | 多文档统一索引 |
| 检索路由 | 无 | 按问题中公司名自动路由到对应文档 |
| UI 展示 | 单文档下拉选择 | 多文档选择 + 对比展示 |

### 2. 表格解析增强

| 对比项 | 工单2 | 工单3 |
|--------|-------|-------|
| 表格标题检测 | 无 | 扫描表格上方文本块，匹配表格标题关键词模式 |
| 列类型识别 | 无 | 识别数字列/文本列/日期列等 |
| 跨页表格合并 | 无 | 检测跨页表格，自动合并内容 |
| 输出格式 | 表格变成混乱文本流 | 结构化标记：`【表格标题】` / `【表格数据】` / `【表格信息】` |
| 关键数值提取 | 无 | `【关键数值】` 段落，把表格中关键数值单独提取出来 |

**chunk 示例对比：**

```
工单2（混乱）：
"军品销售额变动率 营业利润 营业利润 营业利润..."

工单3（结构化）：
"【表格标题】单位：万元
【表格数据】
| 军品销售额变动率 | 2019年1-6月 | 2018年度 | ...
| 1% | 36.58 | 29.23% | 132.85 | 2.58% | ...
【表格信息】表格(4行, 列: 军品销售额变动率...)
【关键数值】
军品销售额变动率=1% | 营业利润变动额=36.58 | 营业利润变动率=29.23%"
```

### 3. 表格感知检索（核心优化）

| 参数 | 配置值 | 作用 |
|------|--------|------|
| `表格权重提升` | 0.3 | 表格类型 chunk 额外加分 |
| `列头匹配提升` | 0.4 | 查询词匹配到表格列名时加分 |
| `表格标题匹配提升` | 0.5 | 查询词匹配到表格标题时加分 |

**检索流程对比：**

```
工单2：FAISS向量 → BM25 → RRF融合 → 重排序(关键词+标题+页码)
工单3：FAISS向量 → BM25 → RRF融合 → 重排序(关键词+标题+页码
                                       + 列头匹配↑0.4
                                       + 表格标题匹配↑0.5
                                       + 表格类型权重↑0.3)
```

### 4. 多文档智能路由（新增）

| 功能 | 说明 |
|------|------|
| 公司名识别 | 从问题中提取公司实体名称 |
| 文档匹配 | 将公司名与两本招股书的标题/内容进行匹配 |
| 路由策略 | 命中公司A的文档 → 优先从文档A检索 |
| 降级策略 | 无法识别的公司名 → 跨文档统一检索 |

### 5. 列名感知匹配

| 步骤 | 说明 |
|------|------|
| 提取列名 | 从 `【表格数据】` 的第一行提取列名列表 |
| 计算匹配 | 统计查询词与列名的重叠词数 |
| 权重加成 | 每匹配到一个列名，结果分 +0.4 |
| 示例 | 查询"营业收入" → 匹配到表格列"营业收入" → 该表格chunk加权 |

## ❌ 仍存在的问题

| 问题 | 原因 | 建议 |
|-----|------|------|
| DeepSeek API Key 已过期 | 配置写死了旧 key | 改成通义千问 qwen-plus |
| bge-small 维度低 | 512维对表格文本区分度不够 | 换 bge-base-zh-v1.5（768维） |
| 表格跨页合并不完善 | 部分表格在页眉/页脚处识别失败 | 增强页眉页脚过滤逻辑 |
| 列名提取依赖表格格式 | 非 markdown 格式的表格列名可能漏提取 | 增加正则回退策略 |
