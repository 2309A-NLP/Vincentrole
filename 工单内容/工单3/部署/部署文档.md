| 项目 | 要求 |

## 3. 部署步骤

### 3.1 安装依赖

```bash
cd 工单3/研发
pip install -r requirements.txt
```

**依赖清单：**

| 包名 | 用途 |
|------|------|
| streamlit | Web 前端框架 |
| pymupdf (fitz) | PDF 文本提取 |
| faiss-cpu | 向量索引与检索 |
| sentence-transformers | BGE Embedding 模型 |
| numpy | 数值计算 |
| rank-bm25 | BM25 关键词检索 |
| requests | LLM API 调用 |

> **注意：** sentence-transformers 首次运行时会自动下载 BAAI/bge-small-zh-v1.5 模型（约 150MB）。如需离线部署，可提前下载到本地缓存目录。

### 3.2 配置 API

编辑 `config.py`，修改 LLM 配置：

```python
LLM_CONFIG = {
    "提供商": "dashscope",                    # 通义千问
    "模型": "qwen-plus",                      # 或 qwen-max
    "API地址": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "API密钥": "sk-your-api-key-here",
    "最大Token数": 1024,
    "温度": 0.1,
    "超时": 15,
    "重试次数": 2,
}
```

### 3.3 准备 PDF

将 PDF 文件放入 `data/` 目录，并修改 `config.py` 中的默认路径：

```python
DEFAULT_PDFS = [
    os.path.join(DATA_DIR, "招股说明书1.pdf"),
    os.path.join(DATA_DIR, "招股说明书2.pdf"),
]
```

> 工单3 支持同时加载**多份 PDF**，在列表中添加即可。

### 3.4 启动系统

```bash
# 方式一：直接运行
streamlit run app/ui.py

# 方式二：通过启动脚本
python run.py
```

Streamlit 会自动分配一个可用端口，终端会显示访问地址（如 `http://localhost:8501`）。

## 4. 首次使用流程

```
1. 打开浏览器 → http://localhost:端口号
2. 系统自动加载默认 PDF（data/招股说明书1.pdf + 招股说明书2.pdf）
3. 如未缓存索引，自动解析PDF → 分块 → 向量化 → 构建索引（约2-3分钟）
4. 索引构建完成后，在输入框输入问题
5. 系统执行：多文档路由 → 表格感知检索 → 重排序 → LLM生成 → 返回结果
```

### 首次加载耗时参考（双文档）

| 阶段 | 耗时 |
|------|------|
| PDF 结构化解析（2本） | ~20s |
| 语义分块（~1500+个chunk） | ~5s |
| BGE 向量化 | ~50s |
| 构建 FAISS + BM25 索引 | ~2s |
| **总计** | **~75s** |

> 后续启动直接从缓存加载索引，耗时 < 1s。

## 5. 配置说明

### 5.1 PDF 解析配置

```python
PDF_CONFIG = {
    "使用引擎": "pymupdf",      # pymupdf 或 pypdf
}
```

### 5.2 分块配置

```python
CHUNK_CONFIG = {
    "分块大小": 800,             # 最大 chunk 字符数
    "分块重叠": 150,             # 滑动窗口重叠字符数
}
```

> 🛠 **调优建议**：表格密集的文档建议 `500/80`；文字连续的文档建议 `1000/200`

### 5.3 Embedding 配置

```python
EMBEDDING_CONFIG = {
    "模型名称": "BAAI/bge-small-zh-v1.5",  # 替换为 bge-base-zh-v1.5 可提升精度
    "设备": "cpu",                            # cpu 或 cuda
    "归一化": True,
    "维度": 512,                              # small:512, base:768
}
```

> 🛠 **调优建议**：将 `bge-small-zh-v1.5` 替换为 `bge-base-zh-v1.5`（768维，需删除旧索引重建）

### 5.4 检索配置（表格感知）

```python
RETRIEVAL_CONFIG = {
    "检索数量": 5,                # 最终返回结果数
    "相关性阈值": 0.0,
    "查询扩展": True,            # 同义词扩展开关
    "重排序数量": 5,             # 重排序 top_k
    "表格权重提升": 0.3,         # 表格chunk额外权重
    "列头匹配提升": 0.4,         # 查询词匹配到列名时的权重
    "表格标题匹配提升": 0.5,     # 查询词匹配到表格标题时的权重
}
```

**三个表格感知参数详解：**

| 参数 | 默认值 | 作用 | 建议调整场景 |
|------|--------|------|------------|
| `表格权重提升` | 0.3 | 所有表格类型chunk统一加分 | 表格问题多 → 调高到0.5 |
| `列头匹配提升` | 0.4 | 查询词命中列名时加分 | 精确数值查询多 → 调高到0.6 |
| `表格标题匹配提升` | 0.5 | 查询词命中表格标题时加分 | 汇总查询多 → 调高到0.7 |

### 5.5 UI 配置

```python
UI_CONFIG = {
    "页面标题": "PDF智能问答系统（表格解析优化版）",
    "页面图标": "📊",
```

## 6. 常见问题

### Q1: 系统返回"文档中未找到相关信息"

**原因：** 检索到的文档片段不包含问题所需的答案。
**排查：**
1. 检查 PDF 是否已成功解析
2. 检查问题涉及的内容是否在 PDF 中
3. 尝试调整表格感知权重参数后重新索引

### Q2: 如何切到通义千问？

修改 `config.py` 中的 LLM 配置：
```python
"提供商": "dashscope",
"模型": "qwen-plus",
"API地址": "https://dashscope.aliyuncs.com/compatible-mode/v1",
"API密钥": "sk-your-qwen-api-key",
```

### Q3: 如何添加更多 PDF？

在 `config.py` 的 `DEFAULT_PDFS` 列表中添加路径：
```python
DEFAULT_PDFS = [
    os.path.join(DATA_DIR, "招股说明书1.pdf"),
    os.path.join(DATA_DIR, "招股说明书2.pdf"),
    os.path.join(DATA_DIR, "招股说明书3.pdf"),   # 新增
]
```
然后删除旧索引文件，重启系统重建索引。

### Q4: 如何重新构建索引？

```bash
rm -f data/vector_index.faiss data/chunk_metadata.json data/chunk_metadata_bm25.pkl
```
重启系统，它会自动重建索引。

### Q5: 表格感知参数怎么调？

| 场景 | 建议配置 |
|------|---------|
| 表格问题多 | `表格权重提升=0.5`, `列头匹配提升=0.6` |
| 文字描述多 | `表格权重提升=0.1`, `列头匹配提升=0.2` |
| 精确数值查询 | `列头匹配提升=0.6`, `表格标题匹配提升=0.7` |
| 混合查询 | 保持默认值 |

## 7. 与工单2 的区别

| 功能 | 工单2 | 工单3 |
|------|-------|-------|
| 文档数量 | 单文档 | 多文档 |
| 表格解析 | 纯文本提取 | 结构化表格（标题+列名+数据+数值） |
| 检索优化 | 关键词+标题+页码 | 关键词+标题+页码+列头匹配+表格权重 |
| 表格权重参数 | 无 | 3个独立可调参数 |
| 多文档路由 | 无 | 按公司名路由 |
| 跨页表格合并 | 无 | 自动检测合并 |
