# 工单6 优化点总结

> 工单编号: 人工智能NLP-RAG-混合检索任务

---

## ✅ 已实现的优化（与工单5对比）

### 1. 多模型 Embedding 支持（新增）

| 模型 | 维度 | 来源 | 说明 |
|------|------|------|------|
| bge-small-zh-v1.5 | 512 | HuggingFace | 轻量快速，默认 |
| bge-base-zh-v1.5 | 768 | HuggingFace | 精度更高，推荐 |
| m3e-base | 768 | HuggingFace | 国产 Moka AI 模型 |
| bge-m3 | 1024 | HuggingFace/ModelScope | 多语言，维度最高 |

切换方式：修改 `config.py` 中的 `EMBEDDING_CONFIG["模型名称"]` 为上述 key 即可。

### 2. Whoosh 全文检索（新增）

| 对比项 | 工单5 | 工单6 |
|--------|-------|-------|
| 检索方式 | 仅向量检索 | 向量 + 全文双通道 |
| 全文引擎 | 无 | Whoosh 倒排索引 |
| 布尔查询 | 不支持 | 支持 AND/OR/NOT |
| 短语匹配 | 不支持 | 支持精确短语 |
| 模糊匹配 | 不支持 | 支持编辑距离模糊 |
| 多字段检索 | 仅 text | text + heading + source_file |

### 3. 3 种重排算法（新增）

```
工单5: 无重排
工单6: ┌─ LLM 重排器    — Kimi 逐条评分 (0-10)
        ├─ TF-IDF 重排器 — 余弦相似度快速排序
        └─ CrossEncoder  — bge-reranker-v2-m3 本地模型
```

| 重排器 | 速度 | 精度 | 是否需要联网 |
|--------|------|------|-------------|
| LLM | 慢 | 最高 | 需 API |
| TF-IDF | 快 | 中 | 否 |
| CrossEncoder | 中 | 高 | 否（本地模型） |

### 4. 混合检索模式

| 模式 | 说明 | 配置 |
|------|------|------|
| `vector` | 仅向量检索 | `默认检索模式: "vector"` |
| `fulltext` | 仅全文检索 | `默认检索模式: "fulltext"` |
| `hybrid` | 向量 + 全文融合 | `默认检索模式: "hybrid"` |

### 5. 3 种融合算法

| 算法 | 说明 | 参数 |
|------|------|------|
| `weighted_avg` | 加权平均融合 | `向量权重: 0.5, 全文权重: 0.5` |
| `rrf` | Reciprocal Rank Fusion | `rrf常数: 60` |
| `vote` | 投票融合 | 两路结果按出现次数排序 |

### 6. 完整评估体系（新增）

| 文件 | 说明 |
|------|------|
| `evaluation.py` | `RAGEvaluator` — 自定义问题批量评估 |
| `test_evaluation.py` | 评估测试脚本 |
| `tech_debt_register.md` | 技术债务登记表 |
| `tech_debt_report.md` | 技术债务分析报告 |

## ❌ 仍存在的问题

| 问题 | 原因 | 建议 |
|-----|------|------|
| Milvus API 不兼容 | pymilvus 3.0 索引格式变更 | 切回 FAISS 或升级代码 |
| bge-m3 模型过大 | 1.9GB，下载/加载慢 | 默认使用 bge-base |
| Whoosh 不支持 GPU | 纯 CPU 检索 | 适合小规模文档 |
