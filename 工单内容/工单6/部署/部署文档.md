## 3. 部署步骤

### 3.1 安装依赖

```bash
cd 工单6/研发
pip install -r requirements.txt
```

额外安装 Whoosh：
```bash
pip install whoosh
```

### 3.2 配置 API

```python
LLM_CONFIG = {
    "提供商": "custom",
    "模型": "qwen-plus",
    "API地址": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "API密钥": "sk-your-key",
    "最大Token数": 1024,
    "温度": 0.1,
}
```

### 3.3 选择 Embedding 模型

```python
# config.py 中切换
EMBEDDING_CONFIG = {
    "模型名称": "bge-base-zh-v1.5",   # 可选: bge-small / bge-base / m3e-base / bge-m3
    "设备": "cpu",
    "维度": 768,                       # 需与模型匹配
}
```

### 3.4 选择检索模式

```python
HYBRID_SEARCH_CONFIG = {
    "默认检索模式": "hybrid",      # vector / fulltext / hybrid
    "向量权重": 0.5,
    "全文权重": 0.5,
    "融合算法": "rrf",             # weighted_avg / rrf / vote
    "rrf常数": 60,
}
```

### 3.5 选择重排器

```python
RERANKER_CONFIG = {
    "启用重排": True,
    "默认重排器": "crossencoder",  # llm / tfidf / crossencoder
    "重排数量": 10,
}
```

### 3.6 启动

```bash
streamlit run app/ui.py
```

## 5. 常见问题

### Q1: 如何切换 Embedding 模型？

修改 `EMBEDDING_CONFIG["模型名称"]`，并同步修改 `维度` 参数：
- bge-small → 512
- bge-base → 768
- m3e-base → 768
- bge-m3 → 1024

切换后需删除旧索引，重启重建。

### Q2: 三种检索模式有什么区别？

| 模式 | 适用场景 |
|------|---------|
| vector | 语义匹配，适合完整句子查询 |
| fulltext | 关键词匹配，适合精确术语查询 |
| hybrid | 综合最优，融合语义+关键词 |

### Q3: 三种重排器怎么选？

| 重排器 | 适用场景 |
|--------|---------|
| LLM | 精度要求最高，不介意速度 |
| TF-IDF | 快速轻量，适合大批量 |
| CrossEncoder | 本地运行，精度和速度平衡 |
