# 工单2 优化点总结

> 工单编号: 人工智能NLP-RAG-基于PDF文档的问答系统优化

---

## ✅ 已实现的优化

### 1. PDF 解析优化

| 对比项 | 优化前（工单1） | 优化后（工单2） |
|--------|---------------|---------------|
| 解析方式 | `page.get_text("text")` 纯文本提取 | 结构化提取：标题层级检测 + 段落分割 + 表格识别 |
| 输出结构 | `{"page": N, "text": "..."}` | `{"page": N, "type": "text"/"table", "heading": "...", "text": "..."}` |
| 表格处理 | 表格变成混乱文本流 | 表格独立标注，保留结构信息 |
| 容错 | 无 | 解析失败自动降级为普通模式 |

### 2. 文本分块优化

| 对比项 | 优化前 | 优化后 |
|--------|--------|--------|
| 策略 | 固定500字滑动窗口 | 200-800字语义自适应 |
| 段落边界 | 仅 `\n\n` 拆分 | 检测 `第.*章` / `^[一、二、]` / `^\d+\.\d+` 等标题 |
| 表格保护 | 无 | 表格整体作为独立chunk，不拆分 |
| 结果（548页PDF） | ~1100个chunk | 1358个chunk（更细粒度，语义更完整） |

### 3. 混合检索优化

| 对比项 | 优化前 | 优化后 |
|--------|--------|--------|
| 检索引擎 | 纯 FAISS IndexFlatIP | FAISS向量 + BM25关键词混合检索 |
| 融合方式 | 无 | RRF（Reciprocal Rank Fusion, k=60） |
| 查询扩展 | 简单 rewrite | 同义词扩展（中英文各15+组） |
| 检索维度 | 纯语义匹配 | 语义 + 关键词双通道互补 |

> **效果数据**：表格查询检索分从 0.016 提升至 0.116（+625%）

### 4. 重排序优化

| 对比项 | 优化前 | 优化后 |
|--------|--------|--------|
| 排序逻辑 | 仅按向量内积 | 综合评分算法 |
| 因子 | 无 | 关键词匹配率 + 标题奖励(×0.15) + 页码因子(前10页×1.05) + 表格奖励(+0.1) |

### 5. LLM 生成优化

| 对比项 | 优化前 | 优化后 |
|--------|--------|--------|
| 模型 | DeepSeek（API Key 已过期） | 通义千问 qwen-plus（200RPM，稳定） |
| Prompt | "仅基于文档内容回答" | 强化版：提取表格数据、引用具体数字 |
| 多语言 | 仅中文 | 自动检测中英文，使用对应prompt模板 |
| 重试 | 无 | 自动重试2次 + 错误分类提示 |
| 容错 | 401/超时无提示 | 分类提示（Timeout/401/429/余额不足） |
| 降级 | 返回空字符串 | 返回错误信息 + 演示模式 |

### 6. 全面容错机制

| 场景 | 处理方式 |
|------|---------|
| PDF文件不存在 | 返回 `status: "error"` + 友好提示 |
| 结构化解析失败 | 自动降级为纯文本解析 |
| API超时 | 分类提示 + 自动重试 |
| API错误 | 按HTTP状态码分类提示 |
| API Key为空 | 演示模式 + 模拟答案 |

### 7. 多语言支持（新增）

| 功能 | 说明 |
|------|------|
| 语言检测 | 基于字符比例自动判断中/英文 |
| 界面语言 | 右上角下拉切换（中文/English） |
| 查询处理 | 中英文分别使用不同的意图模式、实体提取、同义词库 |
| Prompt | 中文用"文档中未找到相关信息"，英文用"Information not found in the document" |
| 模拟答案 | 中英文各一套 |

## ❌ 仍存在的问题

| 问题 | 原因 | 建议 |
|-----|------|------|
| 表格数据检索不到 | chunk 分块策略对表格不友好，表头和数据被拆到不同chunk | 改小 `分块大小`(800→500)，`重叠`(150→80) 后重新索引 |
| bge-small 精度有限 | bge-small 只有 512维，表达能力较弱，对长文档检索精度不够 | 换 bge-base-zh-v1.5（768维），需删除旧索引重建 |
| 招股说明书 548页 | 文档过大，1358个chunk 中真正包含数据的chunk 被淹没 | 考虑按章节分段索引，或加大标题权重 |
| RRF 融合权重固定 | 向量和 BM25 的权重硬编码为 k=60 | 改为可配置参数，让用户前端可调 |
| Prompt 不够精准 | 只给了"文档中未找到相关信息"的退路，没给"优先提取表格数字"的引导 | 改 prompt 为"优先从表格片段中提取具体数值" |

## 🔜 下一步优化建议（优先级排序）

1. **调分块参数** → `分块大小: 500`、`分块重叠: 80`，删旧索引重建
2. **换 Embedding 模型** → bge-base-zh-v1.5（768维），提升语义区分度
