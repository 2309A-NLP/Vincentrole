## 3. 部署步骤

### 3.1 安装依赖

```bash
cd 工单2/研发
pip install -r requirements.txt
```

**依赖清单：**

| 包名 | 用途 |
|------|------|
| streamlit | Web 前端框架 |
| pymupdf (fitz) | PDF 文本提取 |
| faiss-cpu | 向量索引与检索 |
| sentence-transformers | BGE Embedding 模型 |
| numpy | 数值计算 |
| rank-bm25 | BM25 关键词检索 |
| requests | LLM API 调用 |

> **注意：** sentence-transformers 首次运行时会自动下载 BAAI/bge-small-zh-v1.5 模型（约 150MB），请确保网络通畅。如需离线部署，可提前下载模型到本地缓存目录。

### 3.2 配置 API

编辑 `config.py`，修改 LLM 配置：

```python
LLM_CONFIG = {
    "提供商": "dashscope",
    "模型": "qwen-plus",
    "API地址": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "API密钥": "sk-your-api-key-here",
    "最大Token数": 1024,
    "温度": 0.1,
    "超时": 15,
    "重试次数": 2,
}
```

**支持的 API 提供商：**

| 提供商 | API 地址 |
|--------|---------|
| 通义千问 (dashscope) | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| OpenAI | `https://api.openai.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| Ollama（本地） | `http://localhost:11434/v1` |

### 3.3 准备 PDF

将 PDF 文件放入 `data/` 目录，并修改 `config.py` 中的默认路径：

```python
DEFAULT_PDF_PATH = os.path.join(DATA_DIR, "招股说明书1.pdf")
```

### 3.4 启动系统

```bash
# 方式一：直接运行
streamlit run app/ui.py --server.port 8505

# 方式二：通过启动脚本
python run.py
```

访问 `http://localhost:8505`

## 4. 首次使用流程

```
1. 打开浏览器 → http://localhost:8505
2. 系统自动加载默认 PDF（data/招股说明书1.pdf）
3. 如未缓存索引，自动解析PDF → 分块 → 向量化 → 构建索引（约1-2分钟）
4. 索引构建完成后，在输入框输入问题
5. 系统执行：检索 → 重排序 → LLM生成 → 返回结果
```

### 首次加载耗时参考（548页PDF）

| 阶段 | 耗时 |
|------|------|
| PDF 结构化解析 | ~10s |
| 语义分块（1358个chunk） | ~3s |
| BGE 向量化 | ~30s |
| 构建 FAISS + BM25 索引 | ~1s |
| **总计** | **~45s** |

> 后续启动直接从缓存加载索引，耗时 < 1s。

## 5. 配置说明

### 5.1 PDF 解析配置

```python
PDF_CONFIG = {
    "使用引擎": "pymupdf",      # pymupdf 或 pypdf
}
```

### 5.2 分块配置

```python
CHUNK_CONFIG = {
    "分块大小": 800,             # 最大 chunk 字符数
    "分块重叠": 150,             # 滑动窗口重叠字符数
}
```

> 🛠 **调优建议**：表格密集的文档建议 `500/80`；文字连续的文档建议 `1000/200`

### 5.3 Embedding 配置

```python
EMBEDDING_CONFIG = {
    "模型名称": "BAAI/bge-small-zh-v1.5",  # 替换为 bge-base-zh-v1.5 可提升精度
    "设备": "cpu",                            # cpu 或 cuda
    "归一化": True,
    "维度": 512,                              # small:512, base:768
}
```

> 🛠 **调优建议**：将 `bge-small-zh-v1.5` 替换为 `bge-base-zh-v1.5`（768维，需删除旧索引重建）

### 5.4 检索配置

```python
RETRIEVAL_CONFIG = {
    "检索数量": 5,                # 最终返回结果数
    "相关性阈值": 0.0,           # 最低分数过滤
    "查询扩展": True,            # 同义词扩展开关
    "重排序数量": 5,             # 重排序 top_k
}
```

### 5.5 UI 配置

UI_CONFIG = {

## 6. 常见问题

### Q1: 提示 "文档中未找到相关信息"

**原因：** 检索到的文档片段不包含问题所需的答案。
**排查：**
1. 检查 PDF 是否已成功解析（左下角显示文档页数和chunk数）
2. 检查问题涉及的内容是否在 PDF 中
3. 尝试调整分块参数后重新索引

### Q2: 提示 "API密钥无效或已过期"

**原因：** `config.py` 中的 API Key 不正确。
**解决：** 更新 `LLM_CONFIG["API密钥"]` 为有效值。

### Q3: API 调用超时

**原因：** 网络不稳定或 API 服务繁忙。
**解决：** 系统会自动重试2次。如持续超时，检查网络连接。

### Q4: 如何重新构建索引？

```bash
rm -f data/vector_index.faiss data/chunk_metadata.json data/chunk_metadata_bm25.pkl
```
然后重启系统，它会自动重建索引。

### Q5: 如何更换 Embedding 模型？

1. 修改 `config.py` 中的 `EMBEDDING_CONFIG["模型名称"]`
2. 修改 `EMBEDDING_CONFIG["维度"]` 为对应模型的维度
3. 删除旧索引文件
4. 重启系统
