# 工单13 — 金融问答系统 Docker 部署文档

## 一、概述

将工单13优化版RAG问答系统打包为 Docker 容器，部署在服务器上提供金融问答服务。

- **镜像名**: `w13-finance-qa:latest`
- **端口**: 8501
- **基础镜像**: python:3.11-slim
- **模型**: bge-small-zh-v1.5 (本地嵌入)
- **LLM**: 通义千问 qwen-turbo（API调用）

---

## 二、构建镜像

```bash
# 在项目根目录执行
docker build -t w13-finance-qa:latest .
```

---

## 三、运行容器

```bash
# 启动（API Key通过环境变量传入）
docker run -d \
  --name finance-qa \
  -p 8501:8501 \
  -e DASHSCOPE_API_KEY="sk-your-api-key-here" \
  -e LLM_MODEL="qwen-turbo" \
  -v finance-qa-data:/app/data \
  w13-finance-qa:latest
```

**参数说明:**
| 参数 | 说明 |
|------|------|
| `-p 8501:8501` | 映射端口（宿主机:容器） |
| `-e DASHSCOPE_API_KEY` | 通义千问 API Key |
| `-e LLM_MODEL` | LLM模型（默认 qwen-turbo） |
| `-v finance-qa-data` | 持久化数据卷（FAISS索引+PDF） |

## 五、数据管理

### 持久化
```bash
# 数据卷位置（默认）
docker volume inspect finance-qa-data

# 手动备份索引
docker cp finance-qa:/app/data/vector_index.faiss ./backup/
```

### 更新数据
```bash
# 替换PDF后需重建索引（在容器内执行）
docker exec -it finance-qa python -c "
from app.rebuild_index import rebuild
rebuild()
"
```


---

## 七、常见问题

**Q: 容器启动后访问报502？**
A: 等待模型加载完成（首次启动约 30-60s）

**Q: 回答显示"未配置API Key"？**
A: 检查 `DASHSCOPE_API_KEY` 环境变量是否传入

**Q: 想换更快的模型？**
A: 启动时加 `-e LLM_MODEL=qwen-turbo`（默认）或 `qwen-plus`

## 九、国内网络注意事项

由于国内无法直接拉取 Docker Hub 镜像，构建时推荐以下方式：

### 方式一：在云服务器上构建（推荐）
```bash
# 云服务器通常有更好的网络，直接执行
docker build -t w13-finance-qa:latest .
```

### 方式二：本地直接运行（无需 Docker）
```bash
cd 工单13/研发
streamlit run app/ui.py --server.port 8501
```
本地已有完整的 conda 环境和模型，直接运行即可。

### 方式三：使用阿里云 ACK 等服务
将代码推送到 GitHub 后，通过阿里云镜像构建服务自动构建。
