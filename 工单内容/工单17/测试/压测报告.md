## 3. 瓶颈分析

### 3.1 延迟分解

| 阶段 | 耗时 | 占比 |
|------|------|------|
| 向量检索 (ES) | ~200ms | 3% |
| 上下文组装 | ~50ms | <1% |
| **LLM推理 (DashScope)** | **~5-6秒** | **90%+** |
| 响应传输 | ~200ms | 3% |

### 3.2 失败原因分析

| 错误类型 | 数量 | 原因 |
|---------|------|------|
| ConnectionError (MaxRetryError) | 26 | DashScope API限流 |
| SSLError | 0 | - |
| Timeout | 0 | - |

**结论**: 所有失败均为外部API限流导致，非RAGFlow代码问题。

### 3.3 瓶颈定位

```
用户请求 → RAGFlow API → 向量检索 → LLM推理 → 响应返回
                         (200ms)   (5-6秒)    
                                    ↑
                              瓶颈所在（外部API）
```

## 4. 优化实施

### 4.1 已实现优化

| 优化项 | 实现方式 | 效果 |
|--------|---------|------|
| 连接池复用 | aiohttp.TCPConnector (50连接) | 减少连接建立开销 |
