## 1. 优化概述

### 1.1 问题诊断

原始系统在20并发下存在以下问题：
- **成功率**: 86.15% (77/556请求失败)
- **P95延迟**: 32.8秒 (目标≤3秒)
- **失败原因**: 高并发下DashScope API连接超时

### 1.2 优化方案

| 优化项 | 实现方式 | 效果 |
|--------|---------|------|
| 连接池复用 | aiohttp.TCPConnector | 减少连接建立开销 |
| 请求限流 | asyncio.Semaphore(10) | 防止连接耗尽 |
| 响应缓存 | LRU Cache (100条目, 5分钟TTL) | 重复问题0延迟 |
| 自动重试 | 最多3次, 指数退避 | 提高成功率 |

### 1.3 优化效果

| 指标 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| 成功率 | 86.15% | **100%** | +13.85% |
| 重试次数 | 0 | 3 | - |
| 缓存命中 | 0% | 0%* | - |

*注：测试使用10个不同问题，无重复，故缓存命中率为0。实际使用中重复问题可获得缓存加速。

## 2. 部署步骤

### 2.1 前置条件

```bash
# 1. 确保RAGFlow服务运行正常
docker ps | grep ragflow

# 2. 确保Python环境
python3 --version  # 需要3.8+
pip3 install aiohttp

# 3. 准备API配置
echo "your-api-key" > /tmp/.ragflow_key
```

### 2.2 部署优化客户端

```bash
# 复制优化代码到目标目录
cp 优化/optimized_client.py /opt/ragflow-optimized/

# 配置参数
export RAGFLOW_BASE_URL="http://localhost:9380"
export RAGFLOW_CHAT_ID="your-chat-id"
export RAGFLOW_MAX_CONCURRENT=10
```

### 2.3 集成到现有系统

```python
from optimized_client import OptimizedRAGFlowClient, OptimizedConfig

config = OptimizedConfig(
    api_key="your-api-key",
    chat_id="your-chat-id",
    max_concurrent=10,
    cache_size=100,
    cache_ttl=300
)

async with OptimizedRAGFlowClient(config) as client:
    result = await client.chat_completion("你的问题")
    print(result["answer"])
```

## 3. 配置调优

### 3.1 并发控制

```python
# 高性能场景（API配额充足）
config.max_concurrent = 20

# 稳定优先场景（API配额有限）
config.max_concurrent = 5
```

### 3.2 缓存配置

```python
# 高重复率场景（FAQ类问题）
config.cache_size = 500
config.cache_ttl = 3600  # 1小时

# 低重复率场景（个性化问题）
config.cache_size = 50
config.cache_ttl = 60  # 1分钟
```

### 3.3 重试策略

```python
# 网络不稳定环境
config.max_retries = 5
config.retry_delay = 2.0

# 网络稳定环境
config.max_retries = 2
config.retry_delay = 0.5
```

|------|------|------|
| 错误率 | >5% | 检查API配额和网络 |
| 缓存命中率 | <10% | 检查问题多样性 |
| 平均延迟 | >30秒 | 考虑更换LLM模型 |

## 5. 已知限制

1. **LLM API延迟**: 当前使用qwen3-max，单次推理15-20秒，无法通过代码优化解决
2. **DashScope连接稳定性**: 偶发连接超时，已通过重试机制缓解
3. **缓存适用性**: 仅对重复问题有效，个性化问题无法缓存
