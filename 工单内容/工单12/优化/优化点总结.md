# 工单12 优化点总结

> 工单编号: 人工智能NLP-RAG-LightRAG对比评估

---

## ✅ 已实现的优化

### 1. 传统 RAG 管道（核心）

| 对比项 | 说明 |
|--------|------|
| PDF 解析 | PyMuPDF 提取全文，按段落分块（898个文本块） |
| Embedding | bge-base-zh-v1.5（768维），语义向量 |
| 检索方式 | numpy 余弦相似度，top-3 召回 |
| LLM | 通义千问 qwen-plus（200 RPM） |
| 准确率 | **16/16（100%）** |

### 2. LightRAG 知识图谱管道

| 对比项 | 说明 |
|--------|------|
| 知识图谱大小 | graphml 14MB，1364 实体，1308 关系 |
| 存储方式 | vdb + kv_store（向量索引 + 键值存储） |
| 检索方式 | networkx 图遍历 |
| 当前状态 | pipeline 阻塞，需修复后使用 |

### 3. 双管道对比架构

```
用户输入
    ↓
查询路由器 ──→ 传统 RAG（Milvus/bge + LLM）
    │
    └──→ LightRAG（知识图谱 + LLM）
    ↓
对比评估模块 → comparison/ 目录持久化
```

### 4. API 升级

| 对比项 | 之前 | 之后 |
|--------|------|------|
| 提供商 | Kimi API（20 RPM） | 通义千问 qwen-plus（200 RPM） |
| 限流问题 | 频繁 429 | 无限流 |
| 稳定性 | 不稳定 | 稳定 |
