## 三、关键发现

### 3.1 LightRAG 的修复效果

修复后（清空output重新构建知识图谱），LightRAG 从 **12.5% → 100%** 的表面对比准确率。图谱规模增长显著：

```
修复前: 1份PDF,  364 chunks,  2,845 nodes,  4,062 edges
修复后: 2份PDF,  957 chunks,  6,890 nodes, 10,400 edges  ← 2.4x
```

### 3.2 传统RAG的隐性缺陷

表面上看传统RAG是"16/16正确"，但详细对比发现：

- **Q3（关联方）：** 传统RAG答成了"赵依军"（受控制法人），实际上是"赵马克（持股42.35%）"——这是典型的**LLM幻觉**
- **Q543（注册资本）：** 传统RAG只找到了2017年的5,225万，错过了2018年增资到5,520万的信息
- **Q531（法定代表人）：** 传统RAG翻遍了全文没找到"程家明"这个信息

原因：传统RAG的向量检索只匹配了Top-5段落丢给LLM，**检索范围有限**。LightRAG通过图结构关联实体，能跨段落整合信息。

### 3.3 两套系统的互补性

| 场景 | 传统RAG更优 | LightRAG更优 |
|------|-----------|------------|
| 简单事实查询（单一数字/名称） | ✅ 快，准 | 也可，但慢 |
| 需要跨段落整合的信息 | ❌ 容易遗漏 | ✅ 图谱实体关联 |
| 多跳推理（A→B→C） | ❌ 需要LLM本身推理 | ✅ 图结构天然支持 |
| 首次构建成本 | ❌ 无需构建 | ⚠️ 数小时 |
| 增量更新 | ✅ 直接重新向量化 | ✅ 增量插入 |
| 覆盖文档范围 | ❌ Top-K限制 | ✅ 全局+局部检索 |

## 四、结论

1. **LightRAG 在修复后达到与修复前同样的表面准确率**（16/16），但在需要跨段落整合、实体关联推断的问题上（如Q3、Q543、Q531）**实际表现优于传统RAG**
2. **传统RAG并非真正的100%准确**——Top-K片段截断导致检索信息不完整，LLM容易产生幻觉
3. **知识图谱质量是LightRAG效果的瓶颈**：API限流导致实体提取失败是之前准确率低的根本原因；缩小chunk size + 增强重试策略 + 确保output干净 = 图谱构建成功的关键
4. **实际部署建议：** 两种模式互补使用——简单事实查询走传统RAG（快），复杂推理走LightRAG（准）
