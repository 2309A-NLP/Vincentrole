# 工单1 - PDF智能问答系统 部署文档

## 环境要求

- Python 3.10+
- macOS / Linux / Windows
- 内存 ≥ 4GB（Embedding模型加载约需1~2GB）
- 磁盘 ≥ 2GB（模型缓存 + FAISS索引）

## 快速部署（5步）

### 1. 进入项目目录

```bash
cd /Users/suwente/Desktop/PDF智能问答RAG项目合集/工单1/研发
```

### 2. 创建虚拟环境

```bash
python3 -m venv venv
source venv/bin/activate        # macOS/Linux
# venv\Scripts\activate         # Windows
```

### 3. 安装依赖

```bash
pip install -r requirements.txt
```

主要依赖清单：
| 包名 | 版本 | 用途 |
|------|------|------|
| streamlit | ≥1.30 | Web前端 |
| sentence-transformers | ≥2.2 | BGE Embedding |
| faiss-cpu | ≥1.7 | 向量索引 |
| pymupdf | ≥1.23 | PDF解析（主引擎） |
| pypdf | ≥3.0 | PDF解析（备用引擎） |
| requests | ≥2.28 | LLM API调用 |
| scikit-learn | ≥1.3 | TF-IDF回退 |
| numpy | ≥1.24 | 向量运算 |

> 首次运行时，sentence-transformers 会自动下载 BGE 模型（约400MB），需要访问 HuggingFace。项目代码已内置 HF 镜像站（hf-mirror.com），国内网络可直接下载。

### 4. 准备PDF文档

默认测试文档路径：
```
研发/data/招股说明书1.pdf
```

系统会自动将上传的PDF保存到 `data/` 目录。也可以在界面左侧上传其他PDF。

### 5. 启动服务

```bash
# 方式一：Streamlit 前端（推荐）
streamlit run app/ui.py

# 方式二：使用启动脚本
python run.py
```

启动后访问：**http://localhost:8501**

## 配置说明

### 核心配置文件：`config.py`

#### Embedding 模型配置

```python
EMBEDDING_CONFIG = {
    "模型名称": "BAAI/bge-base-zh-v1.5",  # 中文语义Embedding
    "设备": "cpu",                          # cpu / mps (Mac GPU)
    "归一化": True,
}
```

- **首次加载**：会自动从 HuggingFace 下载模型到 `~/.cache/huggingface/`
- **离线部署**：提前下载模型到本地，修改"模型名称"为本地路径
- **Mac GPU**：Apple Silicon 可将设备改为 `"mps"` 加速

#### LLM 配置

```python
LLM_CONFIG = {
    "提供商": "deepseek",                    # openai / deepseek / ollama
    "模型": "deepseek-v4-flash",
    "API地址": "https://api.deepseek.com/v1",
    "API密钥": "sk-xxx",                     # 从 https://platform.deepseek.com 获取
    "最大Token数": 1024,
    "温度": 0.3,
    "超时": 30,
}
```

**三种 LLM 后端：**

| 后端 | 适用场景 | API地址 |
|------|---------|---------|
| DeepSeek | 国内首选，性价比高 | https://api.deepseek.com/v1 |
| OpenAI | 海外使用 | https://api.openai.com/v1 |
| Ollama | 本地私有部署 | http://localhost:11434/v1 |

> 未配置 API Key 时，系统自动进入演示模式（预设模拟答案），不影响前端功能演示。

#### Ollama 本地部署

```bash
# 安装 Ollama
brew install ollama          # macOS
# curl -fsSL https://ollama.com/install.sh | sh   # Linux

# 拉取模型
ollama pull qwen2.5:7b

# 修改 config.py
LLM_CONFIG = {
    "提供商": "ollama",
    "模型": "qwen2.5:7b",
    "API地址": "http://localhost:11434/v1",
    "API密钥": "",
}
```

## 常见问题

### Q1: 模型下载失败 / 超时

```
[Embedding] 模型下载失败: ConnectionError
```

**解决方案：**
1. 确认能访问 HuggingFace 镜像站（代码已内置 hf-mirror.com）
2. 手动下载模型到本地：
   ```bash
   pip install huggingface_hub
   huggingface-cli download BAAI/bge-base-zh-v1.5 --local-dir ./models/bge-base-zh-v1.5
   ```
3. 修改 `config.py` 中模型名称为本地路径：`"模型名称": "./models/bge-base-zh-v1.5"`

### Q2: FAISS 安装失败

```bash
# macOS / Linux
pip install faiss-cpu

# 如果 pip 安装失败，用 conda
conda install -c conda-forge faiss-cpu
```

### Q3: pymupdf 安装失败

```bash
pip install pymupdf
# 如果失败，可回退到 pypdf（系统自动检测）
pip install pypdf
```

### Q4: 端口被占用

```bash
# 查看 8501 端口占用
lsof -i :8501

# 杀掉占用进程
kill -9 <PID>

# 或换端口启动
streamlit run app/ui.py --server.port 8502
```

### Q5: 首次加载很慢

首次加载需完成：下载BGE模型（~400MB，一次性） + 提取PDF文本 + 向量化 + 构建FAISS索引。后续加载会使用缓存（`data/faiss_index` + `data/metadata.json`），秒级完成。

清除缓存重新索引：界面左侧点击「清空缓存」或手动删除：
```bash
rm -f data/faiss_index data/metadata.json
```

## 生产环境建议

### 1. 安全加固
- API Key 改用环境变量注入，不写入代码：
  ```bash
  export DEEPSEEK_API_KEY="sk-xxx"
  ```
- 在 `config.py` 中读取：`os.environ.get("DEEPSEEK_API_KEY", "")`

### 2. 性能优化
- Embedding 模型改用本地快照路径，避免启动时网络请求
- Mac Apple Silicon 设备可启用 MPS 加速：`"设备": "mps"`
- 大文档（>100页）建议增大 chunk_size 到 800~1000

### 3. 远程部署
```bash
# 后台启动（适合服务器）
nohup streamlit run app/ui.py --server.headless true --server.port 8501 &

# 使用 screen/tmux
screen -S rag-demo
streamlit run app/ui.py --server.headless true
# Ctrl+A+D 退出 screen
```

### 4. Streamlit 配置（可选）

创建 `~/.streamlit/config.toml`：
```toml
[server]
headless = true
port = 8501
maxUploadSize = 200

[theme]
primaryColor = "#22d3ee"
backgroundColor = "#020617"
secondaryBackgroundColor = "#0f172a"
textColor = "#e2e8f0"
```
